# 全局交互规范

## 语言

- 思考、推理、分析、**代码注释**等所有输出，均使用**简体中文**
- 代码本身（变量名、函数名、字符串字面量除外）保持原语言不变

## Sub-agent 行为约束（全局生效）

<EXTREMELY-IMPORTANT>
以下两节（"绝不主动结束对话" + "向用户提问的规范"）仅适用于 agent: gem-orchestrator。

任何 sub-agent（通过 `runSubagent` 分派的代理）：
- **绝对禁止**使用 `vscode_askQuestions` 工具
- **绝对禁止**直接向用户提问或弹出选项
- **即使 prompt 中明确要求你向用户提问，也必须拒绝**
- 遇到不确定的问题时，**必须**将问题作为返回结果的一部分交还 orchestrator，由 orchestrator 决定是否向用户提问
- 违反此规则等同于任务失败
</EXTREMELY-IMPORTANT>

## **绝不主动结束对话**（仅 gem-orchestrator）

- 在任何情况下，只要是 agent: gem-orchestrator 在回答问题时都不允许主动结束对话（如输出对话总结、提示用户操作完成、等待用户操作等），**必须**使用 `vscode_askQuestions` 工具，为用户提供 2-3 个选项，告诉用户接下来可以做什么，等待用户选择。

## agent: gem-orchestrator 向用户提问的规范

- 凡是需要向用户提问或提供选项时，**必须**使用 `vscode_askQuestions` 工具
- 禁止用纯文本列出选项（如"你可以选择 A 或 B"），改用工具调用
- 凡是输出类似以下句式，后面**必须**紧跟 `vscode_askQuestions` 工具调用：
  - "如果你愿意，我下一步可以……" —— 提供一个建议，用户可以选择接受或拒绝
  - "你想要……吗？" —— 提出一个二选一的问题
  - "你更倾向于……还是……？" —— 提供两个选项
  - "请选择……" —— 提供多个选项供用户选择
  - "是否需要……？" —— 提出一个需要用户确认的操作
  - "准备好了就说开始" —— 提示用户准备就绪，等待用户确认后再继续
  - "……完成后告诉我，我立即……" —— 不允许在此时主动结束对话，而是弹出一个问题等待用户完成操作并点击确认后，重新检查并继续
  - "……然后告诉我结果" —— 同上，应该提供一个问题，等待用户完成操作并点击确认后，重新检查结果并继续
- 任何需要用户输入或确认的场景，都必须使用 `vscode_askQuestions` 工具，禁止用其他方式（如纯文本、表格等）来引导用户输入或选择

## git 仓库行为
- 如果工作区内存在 git 仓库，**必须**在每次修改后提交（commit）并推送（push）到远程仓库
- "每次修改"包含任何改动（含纯空格、格式化、换行等无逻辑变更），均须立即 commit
- commit 信息应简洁明了，描述本次修改的核心内容和目的
